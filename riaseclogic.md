# Логика RIASEC

## 1. Как считать результат на сайте

### 1) Зашить ключ теста

Для вопросов 1–10 у тебя структура уже идеально ложится на RIASEC:

Сделай фиксированное соответствие:

- **a → R (Realistic / Реалистичный)**
- **b → I (Investigative / Исследовательский)**
- **c → S (Social / Социальный)**
- **d → C (Conventional / Системный)**
- **e → A (Artistic / Артистичный)**
- **f → E (Enterprising / Предпринимательский)**

Это одинаково для всех первых 10 вопросов.

---

### 2) Алгоритм на бэке/фронте

Псевдологика:

```jsx
const map = { a: 'R', b: 'I', c: 'S', d: 'C', e: 'A', f: 'E' };

const scores = { R: 0, I: 0, A: 0, S: 0, E: 0, C: 0 };

answersQ1toQ10.forEach(answer => {
  const type = map[answer];   // например, "a" → "R"
  scores[type] += 1;
});

// сортируем по убыванию
const sorted = Object.entries(scores)
  .sort((a,b) => b[1] - a[1]);  // [ ['A',6], ['S',4], ... ]

const top1 = sorted[0]; // ['A', 6]
const top2 = sorted[1]; // ['S', 4]
const top3 = sorted[2]; // ['I', 3]

const riasecCode = top1[0] + top2[0] + top3[0]; // "ASI"
```

**Важно:**

- Если у нескольких типов одинаковый балл → показываешь несколько кодов (например, A=6, S=6 → "AS/SA").
- Для простоты можешь всегда брать **TOP-3**, даже если 2-й и 3-й равны.

**Логика вывода:**

```jsx
const top2Code = top1[0] + top2[0]; // например, "AS"

let majors = [];

if (majorsByCombo[top2Code]) {
  majors = majorsByCombo[top2Code];
} else if (majorsByCombo[top2Code.split("").reverse().join("")]) {
  majors = majorsByCombo[top2Code.split("").reverse().join("")];
} else {
  // если комбинированного нет — берём объединение по двум типам
  majors = [
    ...majorsByType[top1[0]],
    ...majorsByType[top2[0]]
  ];
}
```

## 3. Как показывать результат на сайте (UX)

После нажатия «Завершить тест»:

1. **Блок 1 — Код и короткое описание**

> Ваш тип: A-S-I (Артистичный — Социальный — Исследовательский)
>
> Вы любите креатив, общение с людьми и разбор сложной информации.

2. **Блок 2 — Подходящие специальности**

Список 10–20 спецов из `majors` (как выше), в виде карточек/чипов.

3. **Блок 3 — Подходящие профильные предметы**

Используешь ответы по вопросу 11 (профильные предметы) как:

- Если выбранные профильные предметы **совпадают** по логике с типом → пишешь:

    "Ваш выбор профильных предметов хорошо согласуется с типом личности, это усиливает результат теста."

- Если **не совпадают** → мягко:

    "Текущий выбор профильных предметов отличается от вашего профиля. Можно обсудить это с профориентатором."


Для реализации:

```jsx
const entProfilesByType = {

```

## 4. Минимальный MVP по шагам

1. **Зашить** в код:
    - mapping a–f → RIASEC
    - словарь `majorsByType`
    - словарь `majorsByCombo` (5–10 самых частых комбинаций достаточно на старт)
2. **Написать функцию `calculateRiasec(answers)`**, которая:
    - принимает массив ответов [ 'a','c','e',... ]
    - возвращает `{ code: "ASI", scores: {...}, majors: [...] }`
3. **Подключить эту функцию к форме**:
    - пользователь заполняет 10 вопросов
    - жмёт кнопку
    - на фронте всё считается и сразу рендерится результат (можно без бэка).

## 5. Генерация PDF с результатами

После завершения теста генерируется PDF-файл со следующей структурой:

1. **Титульная страница**
   - Логотипы университетов
   - Имя студента
   - Дата прохождения теста

2. **Ваш тип RIASEC**
   - Код (например, A-S-I)
   - Описание типа личности
   - Процентное соотношение каждого типа

3. **Рекомендуемые специальности**
   - Список 10-20 специальностей
   - Краткое описание каждой

4. **Профильные предметы**
   - Выбранные предметы ЕНТ
   - Анализ соответствия

5. **Следующие шаги**
   - Контактная информация для консультации
   - QR-код для записи на профориентацию
